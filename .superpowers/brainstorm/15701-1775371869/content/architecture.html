Key Design Decisions

Event store is append-only. No UPDATE or DELETE on events. Every change is a new event. Full audit trail by default.
Projections are disposable. They can be rebuilt from the event log at any time. This means schema changes don't lose history.
PMS gets a special event type. PMSSnapshotRecorded stores the full snapshot as payload. The projection engine knows to treat it as "override" not "accumulate".
AI never writes directly to event store. Extracted data goes to a staging table. User confirms → events are written. Human-in-the-loop is enforced by architecture.
