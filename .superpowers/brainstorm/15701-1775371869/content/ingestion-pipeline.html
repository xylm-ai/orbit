ORBIT — AI Ingestion Pipeline

Section 3 of 5 · From raw document to confirmed events

Pipeline Overview

1
Document Arrives
Via email (Postmark webhook) or manual upload. Stored in S3/object storage. documents row created with status = pending. Celery task enqueued.
2
Document Classification
LLM reads first page / metadata and classifies: contract_note | pms_transaction | cas | bank_statement. Assigns to correct entity + portfolio if identifiable (PAN, account no, provider name).
3
Pre-processing
PDF → text via pdfplumber. Scanned PDFs → OCR via Tesseract. Excel → pandas. Tables detected and extracted as structured grids. Text cleaned and chunked for LLM context limits.
4
LLM Extraction (GPT-4o)
Structured prompt per document type. Extracts: ISIN, security name, quantity, price, amount, date, transaction type. Returns JSON with per-field confidence_score. Low-confidence fields flagged for human review.
Contract Note → SecurityBought/Sold
PMS Txn → SecurityBought/Sold/Dividend
CAS → MFUnitsPurchased/Redeemed
5
Normalization
ISIN lookup + validation. Security name → canonical name mapping. Currency normalization. Date parsing (DD/MM/YY, DD-Mon-YYYY etc). "Qty" → quantity, "Mkt Val" → current_value. Duplicate detection (same ISIN + date + amount already in event store).
6
Staging + Validation
Extracted rows written to staged_extractions. Auto-checks: totals reconcile, no missing fields, no duplicate events. Document status → awaiting_review. User notified.
7
User Review & Confirm
UI shows extracted rows. Low-confidence fields highlighted. User can edit inline or reject individual rows. On confirm → events written to portfolio_events. Projections rebuilt. Document status → ingested.

Bank Statement Path (PMS Reconciliation)

Extraction
Bank statement parsed → list of debits + credits with date + amount + narration.
Written as BankEntryRecorded events (not holdings events).
Narration parsed by LLM to detect investment-related entries.
Reconciliation Run
After both PMS transactions + bank entries are ingested:
Match bank debits → SecurityBought (date ± 2d, amount ± 0.5%)
Unmatched entries → ReconciliationFlagged event
User reviews flags and resolves.

Error & Retry Handling

LLM failure / timeout
Celery retries 3x with backoff. After 3 failures → document marked failed, user notified to re-upload.
Low confidence (<0.7)
Field flagged in review UI with yellow highlight. User must explicitly confirm or correct before ingestion proceeds.
Duplicate detected
Event with same portfolio_id + event_date + isin + amount already exists → row skipped with warning shown in review UI.
